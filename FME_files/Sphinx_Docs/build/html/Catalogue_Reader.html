Catalogue_Reader

Documentation du Custom Transformer FME

Description

This Custom Transformer enables to read a CKAN or a CSW server and extract the requested metadata records. Once the metadata records are extracted, they are transformed from their hierarchcal structure (JSON for CKAN and XML for CSW) into a flattened FME attribute and list features. The parameters allow great flexibility on how the CKAN or CSW server are requested.

Note: No modification other then the JSON/XML flattening is performed on the metadata records read from the CKAN/CSW server.

Input Ports

  • Input: One feature to signal the start of the metadata server extraction.

Output Ports

  • Résultats: The features resulting from the CKAN or the CSW server extraction. One output feature for each metadata record read.
  • Métadonnées invalides: Used to output ID not found in the CKAN or CSW server. On ly used when the parameter IDENTIFIANT is used.
  • UUID_invalide: Used when the ID is not a valid UUID. On ly used when the parameter IDENTIFIANT is used.

Parameters

  • SOURCE_URL: The source URL (web address) of the target CKAN or CSW server. There are 6 pre-defined source url and one user defined address (drop down bix):
    • FGP-STAGING_INTERNE: Address of the external internal FGP staging server;
    • FGP-STAGING-EXTERNE:Address of the internal internal FGP staging server;
    • FGP-PROD_INTERNE: Address of the internal FGP production server ;
    • FGP-PROD-EXTERNE: Address of the externall FGP production server;
    • OPENDATA-STAGING: Address of the Open Data staging server;
    • OPENDATA-PROD: Address of the Open Data production server;
    • PYCSW-INTERNAL: Address of the internal PYCSW server;
    • CUSTOM-CSW/CKAN: Custom URL address of a CKAN or CSW server to be supplied by the user in the CUSTOM_URL parameter.
  • CUSTOM_URL: The address of the server to be used when CUSTOM-CSW/CKAN option is choosen with SOURCE_URL parameter.
  • CKAN_FILER: String to be used with CKAN server to filter the result. Ex.: fq=collection:fgp where fq is the CKAN reserved word for filtering, collection is a the name of a key in the JSON document and fgp is the requested value for the filtering;
  • CUSTOM_CSW_SCHEMA: Drop down box specifying the output schema to use to generate the metadata records (default: http://www.iso tc211.org/2005/ gmd). Used with CSW server.
  • CUSTOM_CSW_VERSION: Drop down box specifying the version of the CSW server to use (default: 2.0.2)
  • CUSTOM_URL_CSW_TYPENAMES: Dropdown box specifying which entities must be queried from the information model (default: gmd:MD_Met adata). Used with CSW server.
  • CUSTOM_CSW_LIST_NAME: Dropdown box specifying the name of the FME list attribute used to explode the list attribute . (fefault MD_Metadata{}) Used with CSW server.
  • IDENTIFIANT: UUID identifying the ID of the metadata record to extract from the server. Used with CSW and CKAN server.
  • CUSTOM_URL_BY_ID_ATT_NAME: Not used any more.
  • CSW_ANY_TEXT_FILTER: A string to be used to filter the metadata records. Any string canbe used. can filter any field of the metadata record. Used with CSW server.

Note: For more information on the use of the parameters of the CSW server, refer to the official OGC documentation OGC PDF documentation.

class Catalogue_Reader.ManageHttpCall[source]

Bases: object

This class enables the call to CKAN servers and the creation of FME features with CKAN information.

The results from the CKAN servers are stored in the FME Feature attribute (_response_body). For each FME Feaure the result from the CKAN reading and the JSON transformation is stored in the FME Feature attribute _status_code and the description of the result code in the attribute _status_code_attribute.

CKAN server extraction options:

  • to read only one record from the CKAN server write the UUID of the record to read in the FME Feature attribute _id;

  • to read all the records leave the FME attribute ‘_id’ empty

  • to read specific records use the CKAN filter, write the CKAN filter in the FME Feature attribute _OpenMapsFilter; the format of the CKAN Filter is: fq=collection:federated where fq is a keyword of the CKAN http get request; collection is a keyword of the JSON record and federated is the requested value.
