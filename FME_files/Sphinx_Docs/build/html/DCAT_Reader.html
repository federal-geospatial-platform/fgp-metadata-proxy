DCAT_Reader

Documentation du Custom Transformer FME

Description

This Custom Transformer is used to read ESRI GeoHub DCAT server. GeoHub DCAT reader is using two server extractions (a primary and a secondary) to extract a complete set of DCAT metadata for each record. Metadata extracted from the primary server extracttion are prefixed wirh DCAT_P whereas the the metadata extracted from the secondary server extraction are prefixed with DCAT_S. In order to join the metadata from the primary and the secondary extraction both extraction must share a common identifier when no common identifier are found the is no secondary metadata with the prfix DCAT_S.

Note: There is no option to read only one record with a specific identifier.

Input Ports

  • Input: One feature to start the extraction of all the records from the GeoHub DCAT server.

Output Ports

  • Résultats: All the records read from the GeoHub DCAT server.

Parameters

  • GEO_HUB_SERVICE: Name of the DCAT servers to use. Choices are:
    • Manitoba GeoHub: To read Manitoba GeoHub;
    • Saskatchewan GeoHub: To read Sakatchewan GeoHub;
    • Custom GeoHub: To enter specific values of primary and secondary url;
  • CUSTOM_PRIMARY: Definition of the primary Geo Hub service. Used when the option Custom GeoHub is selected;
  • CUSTOM_SECONDARY: Definition of the seconary Geo Hub service. Used when the option Custom GeoHub is selected. It is possible to leave this field empty, in that case no secondary service will be read and less metadata will be extracted.

DCAT_Reader

class DCAT_Reader.ManageHttpCall[source]

Bases: object

This class enables the call to DCAT GeoHub servers and the creation of FME features with DCAT information.

To extract all the information the DCAT reader must read into a primary and a secondary server. The results from the DCAT servers are stored in the FME Feature attributes (_response_body_P and _response_body_S). For each FME Feaure the result from the DCAT reading is stored in the FME Feature attribute _status_code and the description of the result code in the attribute _status_code_attribute.
