EMPTY_FORMAT_MAPPER_NG

Documentation du Custom Transformer FME

Description:

Custom transformer used to fill the resources{}.format when the format is missing or empty. The custom is using the corresponding resources{}.url to extract the format.

In a first step, the custom transformer will analyse the URL:

  • if the url contains the substring "service=wms" the format is WMS
  • if the url contains the substring "mapserver" or "featureserver" the format is ESRI REST
  • if the url ends with a file format extension (ex.: .shp) it takes the extension as the file format; except if the extension is .zip where it tries to find the original file format before being zipped (like CSV for the following url: blablabla.csv.zip)

In a second step, the custom transformer will validate that the format extracted with the preceding rules is a valid format by checking if the format is present in the table Format.csv.

In a third step, if the format is still unknown (format="other"), the custom transformer extract the MIME-content of header of the URL. The MIME-content is than used to extract the format from the CSV file Mime_type.csv

Parameters

  • CSV_FORMAT: Name of the CSV table containing the list of valid fromat for the resources.
  • CSV_MIME_TYPE: Name of the CSV table containing the MIME-type and the corresponding format.
  • IN_CSV_LOOKUP_TABLES_DIR: Name of the directoty containing the lookup tables.
  • P-T_ABBR: Two letter abbreviation of the province (ex: ON for Ontario)
  • URL_VALIDATION: Flag used to extract the header of the URL, can take two values: Yes: Extract the header in order to get the MIME-type; No: Do not extract the header.

Input Ports

  • DATA_INPUT: The input features to process

OUTPUT Ports

  • DATA_OUTPUT: The processed output features
  • UNKNOWN_MIME_TYPE: The features with a MIME-TYPE not present in the MIME-type CSV file.
